Introduction

pimphand/gdrive connects Laravel to Google Drive. Upload, download, list, and delete files through a small API similar to S3. OAuth handles auth; uploads can stream straight to Drive without touching your server disk.

Requires PHP 8.2+, Laravel 11–13.

Quick start

  1. composer require pimphand/gdrive:@dev
  2. php artisan vendor:publish --tag=gdrive-example
  3. Copy gdrive.env.example values into .env
  4. Set redirect URI in Google Cloud Console
  5. php artisan serve → visit /gdrive → connect Drive

Routes from the example load automatically. You don't need to edit routes/web.php.

Installation

From Packagist

composer require pimphand/gdrive:@dev

Local path (development)

{
  "repositories": [{ "type": "path", "url": "../gdrive" }],
  "require": { "pimphand/gdrive": "^1.0" }
}

Google Cloud

  1. console.cloud.google.com → enable Drive API
  2. Credentials → OAuth client ID → Web application
  3. Add redirect: http://localhost:8000/auth/google/callback

Environment

GDRIVE_CLIENT_ID=
GDRIVE_CLIENT_SECRET=
GDRIVE_REDIRECT_URI=http://localhost:8000/auth/google/callback

GDRIVE_APP_FOLDER=pimpdrive
GDRIVE_TOKEN_ENCRYPTION_KEY=

Run php artisan gdrive:seed-config to check your setup.

PHP

use Pimphand\GDrive\Facades\GDrive;

$file = GDrive::put(
    fileName: 'report.pdf',
    body: fopen($path, 'r'),
    mimeType: 'application/pdf',
    sizeBytes: (string) filesize($path),
);

return GDrive::get($file['id']);
GDrive::delete($file['id']);

Save $file['id'] in your DB. All physical files sit in one Drive folder per account (pimpdrive by default).

JavaScript

Browser uploads go through three steps: init session → send chunks → sync list.

POST /gdrive/api/upload/init
POST /gdrive/api/upload/chunk   (repeat until complete)
POST /gdrive/api/sync

Config (from Blade)

window.GDRIVE_DEMO = {
  initUrl: '/gdrive/api/upload/init',
  chunkUrl: '/gdrive/api/upload/chunk',
  syncUrl: '/gdrive/api/sync',
  chunkSize: 2 * 1024 * 1024,
  csrf: document.querySelector('meta[name="csrf-token"]').content,
};

Upload one file

async function uploadFile(file) {
  const h = { 'Content-Type': 'application/json', 'X-CSRF-TOKEN': GDRIVE_DEMO.csrf };

  const session = await fetch(GDRIVE_DEMO.initUrl, {
    method: 'POST', headers: h,
    body: JSON.stringify({ name: file.name, mime_type: file.type, size: file.size }),
  }).then(r => r.json());

  let offset = 0;
  while (offset < file.size) {
    const end = Math.min(offset + GDRIVE_DEMO.chunkSize, file.size);
    const res = await fetch(GDRIVE_DEMO.chunkUrl, {
      method: 'POST',
      headers: { ...h, 'X-Upload-Url': session.upload_url,
        'X-Content-Range': `bytes ${offset}-${end - 1}/${file.size}`,
        'Content-Type': 'application/octet-stream' },
      body: file.slice(offset, end),
    }).then(r => r.json());
    if (res.complete) break;
    offset = end;
  }
}

See public/js/gdrive-demo.js for the full UI (drag-drop, folders, rename).

Chunk requests need X-Upload-Url and X-Content-Range headers. All POST routes require CSRF.

HTTP API

Routes below come from the published example (routes/gdrive.php).

MethodPath
GET/gdriveUI
GET/gdrive/connectOAuth redirect
POST/gdrive/api/upload/initStart upload
POST/gdrive/api/upload/chunkSend chunk
POST/gdrive/api/syncRefresh file list
GET/gdrive/files/{id}/downloadDownload
DELETE/gdrive/files/{id}Trash file

Security

  • Don't commit tokens or .env
  • Encrypt refresh tokens (TokenCrypto)
  • Check OAuth state on callback
  • Stream uploads — don't store temp files on disk
